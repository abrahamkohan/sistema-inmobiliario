Fase 1
Setup por cliente nuevo
Se repite por cada venta — objetivo: menos de 15 minutos
GitHub
  • Crear repo nuevo desde el template (o mirror del repo base)No usar Fork — queda vinculado al repo original públicamente
  • Nombrar el repo con el nombre del cliente: ej. crm-nombre-cliente
Supabase
  • Crear proyecto nuevo en supabase.com
  • Copiar Project URL y anon key → Settings → API
  • Ejecutar scripts/setup.sql completo en el SQL Editor
  • Crear storage buckets: project-media, property-photos
  • Auth → URL Configuration → agregar la URL del cliente como redirect URL
  • Auth → Users → crear primer usuario admin con email del cliente
  • Asignar rol admin al usuario en tabla user_roles
Vercel
  • Importar el repo nuevo del cliente en Vercel
  • Agregar env vars: VITE_SUPABASE_URL y VITE_SUPABASE_ANON_KEY
  • Ejecutar deploy y verificar que levanta sin errores
  • (Opcional) Conectar dominio personalizado del cliente → Settings → Domains

Mantenimiento y actualizaciones
Proceso continuo para todos los clientes activos

⚠️ Limitación importante — leer antes de implementar

Las variables VITE_SUPABASE_URL y VITE_SUPABASE_ANON_KEY son compiladas por Vite en tiempo de build y quedan incrustadas en el JavaScript. No se pueden cambiar desde el frontend en runtime. Cada cliente necesita su propio deploy en Vercel con sus propias variables de entorno. No hay forma de apuntar a otra base de datos sin hacer un redeploy.
